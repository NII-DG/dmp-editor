## 1. はじめに
本マニュアルは、公的研究資金の提供機関から要求されるデータマネジメントプラン（DMP）の作成支援ツール「DMP Editor」の操作手順をまとめたものです。
本システムは GakuNin RDM（GRDM）と連携して動作するため、利用には GRDM のアカウントおよびアクセストークンが必要です。

## 2. 事前準備：GRDM Tokenの取得
ログインに必要なトークンをお持ちでない場合は、以下の手順で発行してください。

1.  **設定画面へのアクセス**
    Webブラウザで GakuNin RDM のトークン設定ページ（`https://rdm.nii.ac.jp/settings/tokens`）にアクセスします。
2.  **新規トークンの作成**
    「パーソナルアクセストークン」の画面にて、「新規トークン」を選択します。
3.  **トークン情報の入力**
    * **トークン名**: 任意の名前を入力します（例: `dmp-editor`）。
    * **スコープ（権限設定）**: **必ず `osf.full_write` にチェックを入れてください**。
        * ※ `osf.full_write` の権限が付与されていない場合、正しく連携できません。
4.  **作成の完了**
    「作成」ボタンをクリックしてトークンを生成し、表示されたトークン文字列をコピーして控えてください。

## 3. ログイン（認証）

1.  **DMP Editor トップページへアクセス**
    DMP Editor を開き、「GRDMとの接続」画面を表示します。
2.  **トークンの入力**
    「GRDM Token」入力欄に、取得したトークンを貼り付けます。
3.  **認証の実行**
    「認証」ボタンをクリックします。入力されたTokenはブラウザのローカルストレージにのみ保存され、GRDMとの疎通に利用されます。

## 4. プロジェクト一覧画面の操作
ログイン後、「DMP Project 一覧」が表示されます。

* **確認**: 作成済みのDMPプロジェクト名、作成日、最終更新日を確認できます。
* **新規作成**: 新しいDMPを作成する場合は、「**新規 DMP Project を作成する**」ボタンをクリックします。
* **編集**: 既存のDMPを編集する場合は、各行の「**編集**」ボタンをクリックします。

## 5. DMPの新規作成・編集
「DMP Project 編集」画面にて、具体的な計画内容を入力・設定します。

### 5-1. 基本情報の入力
以下の必須項目等を入力してください。
* **プロジェクト名**: プロジェクトの名称
* **研究分野**: リストから選択
* **研究概要**: 研究の目的や概要

### 5-2. 研究者（分担者）の管理
「研究者」セクションの「追加」ボタンからメンバーを登録します。
ポップアップ画面にて、氏名、所属機関、部署、職位、e-Rad番号、役割（研究代表者等）を入力し、「保存」を押してください。

### 5-3. 研究データの管理
「研究データ」セクションの「追加」ボタンからデータセット情報を登録します。
ポップアップ画面にて、データ番号、名称、種類、公開設定、管理者名を入力し、「保存」を押してください。

### 5-4. GakuNin RDM プロジェクトとの関連付け
画面下部のプルダウンメニューから、このDMPを紐づける GakuNin RDM 上のプロジェクトを選択します。

### 5-5. 保存
入力内容は画面最下部の「保存」ボタンをクリックして確定させてください。

## 6. DMPの出力（エクスポート）
作成したDMPはファイルとして出力することができます。DMP編集画面にある「**エクスポート**」ボタンをクリックしてください。所定のフォーマット（Excelファイル）でDMPが出力・ダウンロードされます。
